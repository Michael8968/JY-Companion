## ADDED Requirements

### Requirement: 课堂内容结构化摘要

将课堂录音转写为文本并生成分层结构化摘要。

#### Scenario: 录音上传与转写

- **WHEN** 教师上传一节 45 分钟课堂录音到 `POST /api/v1/classroom/sessions`
- **THEN** 系统调用 ASR 服务将录音转为文本（普通话识别准确率 ≥95%）
- **AND** 转写完成后状态更新为 `completed`

#### Scenario: 结构化摘要生成

- **WHEN** 课堂录音转写完成
- **THEN** 系统提取核心概念、逻辑结构、关键案例、教师强调内容
- **AND** 生成大纲视图（分层级标签）和图表视图（概念关系图）
- **AND** 摘要生成时间 ≤10 分钟/课时

### Requirement: 个性化回家学案生成

结合学生画像和课堂内容，动态生成个性化学习方案。

#### Scenario: 学案自动生成

- **WHEN** 课堂摘要生成完成且学生有画像数据
- **THEN** 系统匹配学生知识掌握图谱和历史错题
- **AND** 生成包含复习要点、针对性练习题、拓展阅读、微课链接的个性化学案
- **AND** 学案个性化覆盖度 ≥85%

#### Scenario: 不同层次学生差异化

- **WHEN** 基础巩固型学生和拓展探究型学生获取同一课的学案
- **THEN** 基础型学案侧重核心知识点巩固和基础练习
- **AND** 拓展型学案包含进阶题目和延伸阅读

### Requirement: 疑问点自动识别

分析课堂语音特征，自动标记潜在疑难片段。

#### Scenario: 语速变化检测

- **WHEN** 系统分析课堂转写文本和对应音频时间轴
- **THEN** 识别教师语速显著变慢、重复讲解、学生提问频发的片段
- **AND** 将这些片段高亮标记为疑问点，附带时间戳和上下文
- **AND** 疑点识别召回率 ≥80%

#### Scenario: 学生查看疑问点

- **WHEN** 学生请求 `GET /api/v1/classroom/sessions/{id}/doubts`
- **THEN** 返回按重要度排序的疑问点列表
- **AND** 每个疑问点可跳转到对应的录音片段和摘要段落
