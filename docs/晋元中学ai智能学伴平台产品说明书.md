## 一、产品概述

### 1.1 产品基本信息

| 字段 | 内容 |
|------|------|
| **产品名称** | 星澜中学 AI 智能学伴平台 |
| **产品英文名** | Jinyuan High School AI Intelligent Learning Companion |
| **产品版本** | V1.0（首期建设版本） |
| **核心技术底座** | 华东师范大学「启创·InnoSpark 1.0」教育大模型 |
| **适用对象** | 星澜中学全体在校学生（高中阶段）、教师及家长 |
| **使用场景** | 校内课堂复盘、校外自主学习、移动随时随地辅导 |
| **文档状态** | 正式发布 |

### 1.2 产品背景

根据2025年8月国家发布的《关于深入实施"人工智能+"行动的意见》，明确要求"创新智能学伴、智能教师人机协同教育教学新模式"，并要求"加快实现大规模因材施教"。上海市相继出台《上海市推进实施人工智能赋能基础教育高质量发展的行动方案（2024-2026年）》，将教育列为AI应用重点领域，鼓励建设自适应学习平台与智能助教系统。

星澜中学作为上海市人工智能教育实验校，已建成支撑"选择教育"理念的数字化基座——包括全学科数字资源库、平板电脑终端及近三学年学生数据。然而，随着"选择教育"纵深推进，现有数字化实践难以满足学生个性化、分层化的自主学习需求，亟需引入 AI 智能学伴突破能力边界。

### 1.3 产品目标

- 以华东师范大学「启创·InnoSpark 1.0」教育大模型为支撑，全面覆盖学业辅导、心理陪伴、健康守护等领域，创新智能学伴人机协同教育新模式；
- 实现教育大模型与校本教学资源的深度融合，通过资源精准匹配学生分层特征、功能动态适配学习需求，满足基础巩固型、能力提升型、拓展探究型等不同层次学生的个性化自主学习需求；
- 探索 AI 赋能学生个性化学习与"五育并举"的创新实践，打造可复制、可推广的上海市人工智能教育应用创新实践案例。

## 二、产品定位与设计理念

### 2.1 核心定位

| 定位 | 说明 |
|------|------|
| **是「伴」，非「师」** | 学伴以平等、支持性伙伴角色为首要定位，以鼓励性语气激发学生自信与好奇心，帮助学生在探索过程中实现自主学习，而非居高临下地指导或替代教师。 |
| **是「协作者」，非「替代者」** | 学伴通过智能工具增强教师、家长与学生三方协同能力，作为人机协同的桥梁，帮助家长更深入理解孩子的学习进度，促进更有效的沟通与合作，而非取代人类关系。 |
| **「成长型」思维体** | 学伴具备强大的自适应学习能力，通过分析用户行为、偏好与反馈，动态调整响应策略，不仅适应当前需求，还能预见未来挑战，随用户共同成长，形成持续优化的伙伴关系。 |

### 2.2 四大设计原则

| 原则 | 说明 |
|------|------|
| 🎭 **人格化设计** | 拥有高度稳定且富有亲和力的拟人化角色设定，包括专属名字、风格统一的头像、富有感染力的语音特征，以及始终如一的性格特质（耐心、幽默、积极鼓励），构建可信可爱的角色形象。 |
| 🌐 **场景化适配** | 灵活融入用户多样化生活与学习环境，无论是学校课堂、家庭学习还是户外探索，均能依据具体情境调整交互方式与内容表达，实现无缝场景化陪伴。 |
| ⚡ **主动式交互** | 不仅响应用户直接提问，更基于行为习惯、学习进度及情景状态深度理解，主动发起个性化关怀、适时提醒和有针对性建议，真正实现数据驱动的智能化伴随。 |
| 🛡️ **安全与可信保障** | 严格遵循数据隐私保护规范，确保用户数据处理与存储的安全性与合规性；坚持正向积极的价值观导向，输出安全、健康、有益的内容，建立长期可信的用户关系。 |

## 三、系统架构

### 3.1 五层架构总览

| 架构层级 | 核心组成 |
|----------|----------|
| **应用场景层**（舞台） | 移动应用平台（iOS/Android）、桌面学习环境集成（Windows/macOS/Linux）、沉浸式AR集成、可穿戴设备轻量化交互 |
| **人格交互层**（性格与面孔） | 人格化对话系统、多模态数字形象、情感表达与识别引擎、学伴社区群管理 |
| **能力模块层**（工具箱） | 学业辅导智能体、课堂复盘智能体、情感陪伴智能体、健康守护智能体、创意创作智能体、生涯规划智能体 |
| **智能中枢层**（大脑） | 用户意图理解与分发、多源情景感知与融合、任务规划与智能体协同编排 |
| **基础资源层**（身体与记忆） | 核心AI大脑（教育大模型）、领域知识库、用户数据资产中心、多模态感知与生成引擎 |

### 3.2 核心数据流向

用户通过多终端发起交互请求 → 智能中枢层进行意图理解与情景融合 → 路由至对应能力模块智能体 → 调用基础资源层大模型及知识库处理 → 经人格交互层包装为个性化回复 → 以多模态形式呈现给用户。全链路均有安全审计与隐私保护机制。

## 六、安全与合规说明

### 6.1 数据安全

- 所有用户数据传输均采用 TLS 1.3 加密协议；存储数据采用 AES-256 加密或符合国密标准的 SM4 算法；
- 严格遵循最小化原则，仅采集平台功能所必需的用户数据；
- 用户个人信息不得用于商业广告目的，不得向第三方出售；
- 数据存储服务器部署于中国境内，符合《数据安全法》与《个人信息保护法》相关要求；
- 支持用户数据导出与账号注销，账号注销后30日内完成数据删除。

### 6.2 未成年人保护

- 针对未成年用户开启内容安全过滤，屏蔽任何不适合学生年龄段的内容；
- 情感陪伴模块内置危机预警协议，危机信号触发时优先通知绑定的教师或家长，同时提供专业心理援助资源；
- 健康守护智能体强制实施用眼健康规范，连续使用超过40分钟后强制触发休息提醒；
- 不收集学生面部图像用于身份识别或商业用途；面部情感计算数据仅用于实时情绪判断，不持久存储原始图像。

### 6.3 AI内容安全

- 多维度内容安全规则体系实现实时检测与精准拦截，覆盖违法违规、有害信息、低俗内容等多类风险；
- 价值观合规分类器确保输出内容符合社会主义核心价值观与教育导向；
- 对高敏感话题（如政治、宗教、极端内容）实施严格过滤，引导用户理性客观思考；
- 所有 AI 生成内容均加注明显标识，明确告知用户内容由 AI 生成，避免误导；
- 建立完善的内容申诉与人工审核机制，7×24小时响应处理用户反馈的问题内容。

### 6.4 合规认证要求

| 合规类别 | 要求标准 |
|----------|----------|
| **网络安全** | 符合等保2.0三级认证要求 |

| **边缘节点** | 校园本地服务器：轻量化推理引擎、实时感知计算、离线缓存服务 |
| **客户端** | 移动端App（iOS/Android）、桌面客户端（Win/macOS/Linux）、浏览器插件、AR设备端 |

| **容灾备份** | 关键服务双活部署，数据每日全量备份 + 实时增量备份，RTO ≤4小时，RPO ≤1小时 |
